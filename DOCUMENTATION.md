
Документация по сайту «Филиал №7 ГВКГ им. Н.Н. Бурденко»
1. Общие сведения
Название проекта: Официальный сайт Филиала №7 Главного военного клинического госпиталя имени Н.Н. Бурденко
Тип сайта: Информационно-справочный (статический HTML/CSS/JS)
Целевая аудитория: Военнослужащие, ветераны, члены их семей, гражданский персонал, а также все желающие получить информацию о работе госпиталя.
Задачи сайта:

Предоставление актуальной информации об отделениях, услугах, документах, контактах.

Возможность записи на приём через электронную форму.

Адаптивный интерфейс для всех устройств.

Доступность для людей с ограниченными возможностями (версия для слабовидящих, контрастный режим).

Сайт разработан на чистых технологиях (HTML5, CSS3, JavaScript) и не требует серверной обработки. Может хоститься на любом статическом хостинге, включая GitHub Pages.

4. Инструкция по локальному запуску и редактированию
4.1. Требования
Любой современный браузер (Chrome, Firefox, Edge, Safari).

Для редактирования – любой текстовый редактор (VS Code, Sublime Text, Notepad++).

4.2. Запуск
Скачайте все файлы проекта в любую папку на компьютере.

Откройте файл index.html в браузере (двойным щелчком или перетаскиванием).

Навигация по сайту работает через ссылки в шапке.

4.3. Как вносить правки
Изменение текста: Откройте нужный HTML-файл, найдите соответствующий абзац или заголовок, отредактируйте текст. Сохраните файл и обновите страницу в браузере.

Изменение стилей: Откройте соответствующий CSS-файл в папке css/. Например, для изменения цветов – отредактируйте переменные в reset.css.

Добавление новой страницы: Скопируйте существующую страницу (например, about.html) в папку pages/, измените содержимое блока <main>, а также заголовок <title> и мета-описание. Не забудьте добавить ссылку на новую страницу в навигацию (<nav> во всех HTML-файлах).

Работа с формой записи: Валидация и пошаговость описаны в js/form-validation.js и js/form-wizard.js. При реальном развёртывании нужно заменить функцию submitAppointment() на отправку данных на сервер (например, через AJAX).

6. Рекомендации по дальнейшей поддержке
Регулярно обновляйте документы: файлы лицензий, уставов и приказов (папка docs/) должны быть актуальными. При изменении файла не забудьте обновить ссылку в pages/documents.html.

Изображения: оптимизируйте их перед загрузкой (размер, формат WebP для улучшения скорости). Все изображения должны иметь атрибуты alt для доступности.

Форма записи: сейчас она работает в демонстрационном режиме (имитация отправки). При подключении реального бэкенда нужно:

Убрать alert('Код подтверждения отправлен (имитация). Введите 1234') и реализовать реальную отправку SMS.

В submitAppointment() заменить генерацию талона на отправку данных методом POST на серверный скрипт (например, через fetch).

Валидация номера телефона: в form-validation.js используется проверка формата +7XXXXXXXXXX. Убедитесь, что это соответствует реальным номерам.

Доступность: Регулярно проверяйте контрастность текста и корректную работу панели доступности на разных устройствах.

SEO: При необходимости добавьте в каждую страницу мета-теги og:title, og:description, twitter:card для улучшения шаринга в соцсетях.

7. Технические детали
Версия HTML: HTML5

Версия CSS: CSS3 (с использованием переменных, flexbox, grid)

JavaScript: ES6 (поддерживается всеми современными браузерами)

Кодировка: UTF-8

Адаптивность: Mobile First – медиа-запросы в responsive.css

Кроссбраузерность: Проверено в Chrome, Firefox, Safari, Edge (последние версии). Не поддерживается IE11.

Безопасность: Все внешние ссылки открываются в новой вкладке с rel="noopener".
